信息检索课堂作业0510

发表于 2017-05-16   |  

五月十号信息检索课堂作业

目标：

爬取四川大学公共管理学院师资队伍信息

平台以及工具等：

Mac下使用Scrapy进行爬取，选取元素时选用的是CSS选择器

遇到的问题

  1. 中文乱码

刚开始爬取的数据打开是乱码，当然严格来说也不是乱码，是unicode编码，只是人看不懂，虽然可以转换，但总感觉不是事，后面在官方文档中找到了靠谱的解决方案：

If unset or set to None (default) it uses UTF-8 for everything except JSON output, which uses safe numeric encoding (\uXXXX sequences) for historic reasons.

原因是在所有格式输出中，除了JSON都支持utf-8编码（WTF？？？），需要在setting文件中进行设置

FEED_EXPORT_ENCODING = 'utf-8'

  1. 空格与tab

运行代码时，console一直报错，显示IndentationError: unindent does not match any outer indentation level，我仔细检查了相关行数的代码以及层级关系，发现都没错，我突然想到在Python中空格（spaces）与tab的问题，我比较常用的语言是javascript，虽然关于空格和tab的撕逼经久不衰，但我在写前端的时候是很少遇到的，只是以前在翻看资料时偶然间看到说Python中空格和tab容易出问题，于是，我用文本编辑器（sublime text）将代码中所有tab显示出来并转化为空格，然后运行通过。

代码以及结果

代码思路比较简单，这里就不说了

import scrapy


class AuthorSpider(scrapy.Spider):
    name = 'irclass1'

    start_urls = ['http://ggglxy.scu.edu.cn/index.php?c=article&a=type&tid=18']

    def parse(self, response):
        for staff in response.css('ul.teachers_ul li.fl').extract():
            def extract_with_css(query):
                return response.css(query).extract_first().strip()

            yield {
            'name': extract_with_css('ul.teachers_ul li.fl div.fr h3.mb10::text'),
            'title': extract_with_css('ul.teachers_ul li.fl div.fr p.color_main::text'),
            'depart': extract_with_css('ul.teachers_ul li.fl div.fr div.desc p:first-child::text'),
            'mail': extract_with_css('ul.teachers_ul li.fl div.fr div.desc p:first-child + p::text'),
            'img': extract_with_css('ul.teachers_ul li.fl div.fl > a::attr(href)'),
            }

        next_page = response.css('div.pager.cf.tc.pt10 li:nth-last-child(2) a::attr(href)').extract_first()
        if next_page is not None:
            next_page = response.urljoin(next_page)
            yield scrapy.Request(next_page, callback=self.parse)
