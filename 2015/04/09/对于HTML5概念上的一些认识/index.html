对于HTML5概念上的一些认识

发表于 2015-04-09   |  

初识html5

1、我与HTML5

对于前端来说，HTML5的开发是一个无法避开的领域，同时，整个前端开发也因为HTML5的成熟而充满活力。开始学习前端时，忙于学习基础知识，把HTML5当做新技术，只是浅尝辄止，束之高阁。不过到了现在这个时候，我有必要对HTML5开始一次比较系统的学习，这篇文章起一个头，从我的视角从一个概要的层面来说一些对于HTML5的认识。

2、究竟什么是html5？

认识一个新事物（虽然算起来html5也不小了。。。）首先需要搞清楚它的定义，最常用的方式应该就是把关键词丢到搜索引擎去搜，很多相关网站给出的解释是：

“HTML5 是下一代的 HTML”

虽然初看上去很简洁明了，但对于我来说，当初可是造成了很大的困扰。因为HTML长期给人的一种印象是简单的标记语言，连编程语言都算不上，但现实中遇到的一些实际应用比如离线存储啥的明显就不应该是传统html的内容啊。直到看到MDN的一篇文章才理清了我的思绪，给了我一个全面的html5认识。那就是这句话：

HTML5 is the latest evolution of the standard that defines HTML. The term represents two different concepts:
1、It is a new version of the language HTML, with new elements, attributes, and behaviors,
2、and a larger set of technologies that allows more diverse and powerful Web sites and applications. This set is sometimes called HTML5 & friendsand often shortened to just HTML5.

我的理解为HTML5是HTML这门语言最新演进版，但它涵盖两个方面，其一就是传统意义上的HTML语言版本演进，这是我以前对于html5的认识，比如增加了新的语义元素等，其二也是最重要的一方面，一系列的新兴web技术也被归到HTML5里面，在看《JavaScript高级程序设计》这本书时，我了解到这里面新兴的技术很多是一些新的JS API和规范。换句话说，HTML5极大的不同于以前的HTML，关于JS涉及了很多。

总的一句话就是，HTML5不仅仅是HTML版本的迭代，更是一系列新技术规范。

我对于html5的学习也将大致从这两个方向出发，不过很明显，技术规范这一部分的内容会很多。

2.1、逸事——HTML5还是HTML 5？

最初接触时，真没有发现这两者有什么不同，以为只是印刷或者一种书写习惯罢了。有意思的事，后来发现这两者还是有很细微的区别的。

众所周知，一些浏览器厂商不满W3C，于是成立了WHATWG，初期这个组织算是推进HTML5标准制定的主要动力，在认识到新标准大势所趋之后，W3C基于WHATWG的成果推进新标准，但名字变成了HTML 5，多了一个空格。

来源：https://developer.mozilla.org/zh-CN/docs/Web/Guide/HTML/HTML5/Introduction_to_HTML5

2.2、HTML5与HTML4的区别

这一部分内容算是http://www.w3.org/TR/html5-diff/ 这部分资料的学习总结

  • 语法

    • HTML5的语法兼容了旧版本HTML几乎大部分语法，但对于旧版本的一些不被大部分用户代理支持的语法
    • HTML5的语法要求开发者声明字符编码，有三种实现方法：

    At the transport level; for instance, by using the HTTP Content-Type header.

    Using a Unicode Byte Order Mark (BOM) character at the start of the file. This character provides a signature for the encoding used.

    Using a meta element with a charset attribute that specifies the encoding within the first 1024 bytes of the document; for instance, could be used to specify the UTF-8 encoding. This replaces the need for although that syntax is still allowed.

    • 文档类型声明保证了HTML文档以标准模式渲染文档。HTML5使用<!DOCTYPE html>声明，旧版本的HTML因为牵涉到DTD文档类型声明会更长一点。
    • MathML与SVG这部分我不熟悉，不做阐述
    • 其他内容：这里面只捡我认为比较重要的部分来说。空元素允许有结束斜线；属性之间必须分隔开至少一个空格的字符距离；属性值为空的属性可以省去等号和值，不是布尔值属性也这样

  • 语言

    • 新的元素：为了更好的结构化，一些新的元素被引入

    section表示一个普通的文档或者应用部分，而且应当一起使用h1，h2，h3，h4，h5，h6这些元素来表明文档的结构

    article表示文档内容的独立片段，比如说一篇博客记录或者一片新闻文章。

    main表示一个文档或应用程序的主体的主要内容

    aside表示一块内容是稍微相关的页面的其余部分

    header表示一组介绍或者导航帮助

    footer表示页脚的部分，可以包含关于作者，版权信息等信息

    nav表示文档的一个部分用于导航

    上面的标签应该是HTML5中比较常用的，还有差不多二十多个新标签没介绍，这里就不深入了

    • 新的属性：一些新属性被添加到已经是HTML4的元素中

    html元素有一个称为manifest的新属性，它指向一个与脱机Web应用程序的API一起使用的应用程序缓存清单

    meta元素有一个charset新属性用于规定文档的字符编码

    更多的新属性查看文档

    • 改变的元素：在HTML5中，有部分元素轻微改变了他们的意义来更好的显示它们在web的中的用法，使得他们更加有用（看了这部分我才知道很多元素并没有消亡，而只是改变了意思）

    strong元素现在表示重要性而不是重语气

    small元素表示不重要的评论比如小的打印文字

    hr元素表示一个段落层次的专题停顿

    更多的改变元素查看文档

    • 改变的属性：一些元素属性通过各种方式改变了其原有意义

    form的action属性再也不允许一个空的URL了

    img的alt属性有了更复杂的要求，而且在某些情况下可以省略

    link的href属性不再允许空URL

    img iframe object的width height属性不允许包括百分比，它们也不被允许拉伸到比固有的宽高比不同的比例

    更多改变的属性查看文档

    • 淘汰的元素：这些元素不会被web开发者使用，但用户代理仍会支持它们（这些才是真正消亡的）

    比如isindex basefont big center frame frameset等等

    • 淘汰的属性：部分HTML4中的属性已经在HTML5中被淘汰，用户代理仍会处理那些比较老的文档，但开发者不应该使用

    link的target；html的version；img的name；大部分元素的align；body的background；table的cellspacing和cellpadding等等

  • 内容模型

    内容模型定义了元素如何嵌套－什么才能作为某元素的儿子（或后代）。

    HTML4有主要两个元素种类：”inline”和”block-level”，而它们的内容模型却很复杂。

    HTML5没有使用“inline”，“block-level”术语作为它内容模型的规则，但它有比HTML4更多的种类，一个元素可以什么也不是，也可以是它们中的一个，也可以是它们中的很多类，比如说元数据内容：link``script；流内容：span``div；分块内容：aside section；标题内容：h1；短语内容：span img text，这有点类似HTML4的“inline”；嵌入内容：img iframe svg；交互内容：a button label，交互内容不允许嵌套；

    还有很多更为详细的内容查看文档

  • APIs

    HTML5引入了很多新的APIs，并且也拓展了，改变了或淘汰了一些存在的APIs。大致分为新增加的APIs、改变的APIs、拓展的APIs、淘汰的APIs

    这里面内容我不是很熟，而且要展开来说实在过于庞大，这里就不做进一步阐述了

3、html5的具体内容

要学习html5，需要对html5 的具体内容有一个了解，做到心中有数，由于2014年10月份，W3C正式发布了HTML5的正式推荐标准，相当于html5定稿，因此最全面的内容概述，莫过于W3C的HTML5推荐标准了

3.1、对于W3C文档成熟过程的简单理解

关于这部分内容可以直接戳 http://www.w3.org/2014/Process-20140801/#reports 但这里面的东西实在太多了，只要掌握了解过程即可，在查询了一些资料后我进行了简化。

按照W3C的过程文档，一个推荐标准的发展需要通过不同的成熟级别。

来源：http://zh.wikipedia.org/wiki/W3C%E6%8E%A8%E8%8D%90%E6%A0%87%E5%87%86

  1. 工作草案（WD）——Working Drafts
    工作草案文档是一个标准的首次公开发表，在这一阶段，标准文档很可能会发生重大的修改

  2. 候选推荐标准（CR）——Candidate Recommendations
    候选推荐标准与工作草案相比更加稳定，标准文档还可能发生进一步的修改，但此时重大的特性大部分已经锁定

  3. 提案（建议）推荐标准（PR）——Proposed Recommendations
    在这一阶段，标准文档已经提交到W3C顾问委员会进行最后的批准，这一阶段很少导致对标准的任何重要的修改

  4. W3C推荐标准（REC）——Recommendations
    最成熟的发展阶段，可以广泛应用

  5. 后续的修订
    推荐标准通过分别发布的勘误表来更新，当积累到足够的实质上的修订，会发布推荐标准的一个新版本
